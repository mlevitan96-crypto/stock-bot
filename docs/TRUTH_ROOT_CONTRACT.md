# Truth Root Contract — Canonical paths, schemas, freshness

**Status:** Phase 1 (mirror). CTR is optional; legacy paths remain authoritative for readers until Phase 2.

## 1. Canonical root

- **Default:** `/var/lib/stock-bot/truth`
- **Override:** `STOCKBOT_TRUTH_ROOT` (environment variable)
- **Enable writes:** `TRUTH_ROUTER_ENABLED=1`
- **Mirror legacy:** `TRUTH_ROUTER_MIRROR_LEGACY=1` (default when enabled)

## 2. Directory layout

```
<STOCKBOT_TRUTH_ROOT>/
  execution/     # (future) orders, fills, attribution
  gates/         # expectancy.jsonl — gate truth
  health/        # signal_health.jsonl, signal_score_breakdown.jsonl, freshness.json
  exits/         # exit_truth.jsonl, exit_attribution.jsonl
  telemetry/     # score_telemetry.json, score_snapshot.jsonl
  meta/          # schema_version.json, producer_versions.json, last_write_heartbeat.json
```

## 3. Streams and freshness

| Stream (relative path) | Expected max age (sec) | Schema / notes |
|------------------------|------------------------|----------------|
| gates/expectancy.jsonl | 300 | ts_eval_epoch, symbol, score_used_by_gate, gate_outcome, fail_reason |
| health/signal_health.jsonl | 600 | ts, symbol, components (has_data, source, contribution) |
| health/signal_score_breakdown.jsonl | 300 | ts_eval, symbol, signals[], composite_score_pre/post |
| exits/exit_truth.jsonl | 600 | ts, symbol, exit_pressure, decision, close_reason |
| exits/exit_attribution.jsonl | 600 | Attribution schema v1.0.0 (see ATTRIBUTION_SCHEMA_CANONICAL_V1) |
| telemetry/score_telemetry.json | 600 | scores, components, missing_intel, last_update |
| telemetry/score_snapshot.jsonl | 300 | ts, symbol, composite_score, gates |

## 4. Meta and heartbeat

- **meta/schema_version.json:** `{"schema_version": "1.0.0"}` (written once by router).
- **meta/producer_versions.json:** `git_sha`, `service`, `updated_iso` (written once by router).
- **meta/last_write_heartbeat.json:** Updated on every CTR write. Fields: `ts_iso`, `ts_epoch`, `stream`. EOD must fail if this file is stale when using CTR.
- **health/freshness.json:** Per-stream `last_ts`, `last_mtime`, `expected_max_age_sec`. Updated on each stream write.

## 5. Truth manifest (Phase 2+)

- **meta/truth_manifest.json:** Lists all streams, expected freshness, schema versions. Can be generated by a script or by the router.

## 6. Contract for audits

- **Dashboard truth contract:** When `TRUTH_USE_CTR=1` and `STOCKBOT_TRUTH_ROOT` is set, dashboard truth audit and EOD use CTR paths and validate `meta/last_write_heartbeat.json` freshness.
- **No silent inference:** If CTR is required and heartbeat is missing or stale, EOD and audits MUST fail with a clear message.
