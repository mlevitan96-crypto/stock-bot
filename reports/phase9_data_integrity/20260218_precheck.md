# Data integrity hard gate — pre-execution memo (2026-02-18)

## Multi-model review (before implementing)

| Lens | Question | Note |
|------|----------|------|
| **Adversarial** | What could still lie to us? | (1) high_water could be populated from executor but never updated during the trade (stale). (2) Blame could show unclassified_pct &lt; 100% by loosening rules without fixing data—we must not loosen. (3) Effectiveness could be run on a subset of logs (date range, join failures) and look good while production still lacks data. |
| **Quant** | Are metrics mathematically sound? | Giveback = (MFE - realized) / MFE when MFE &gt; 0; if high_water is missing we fall back to entry_price → MFE = 0 → giveback undefined (null). Blame: weak_entry uses entry_score &lt; 3; exit_timing uses giveback ≥ 0.3 or MFE &gt; 0 with loss. Both require data on joined rows; without entry_score/giveback, 100% unclassified is correct. |
| **Product** | Are outputs interpretable and auditable? | Every metric must be explainable; no silent 0/0 or N/A without reason. We will: (1) ensure exit_quality_metrics are emitted when high_water is available; (2) surface unclassified_pct explicitly; (3) add a guard log when high_water is unavailable so we can count how often giveback cannot be computed. |

## Scope

- **In scope:** Trace exit quality end-to-end on droplet; minimal fix to populate info["high_water"] before log_exit_attribution; trace blame and ensure unclassified explicit; re-run baseline v3; sign-off.
- **Out of scope:** No tuning, overlays, or paper run changes; no loosening of classification rules; no backtest overlay measurement.
