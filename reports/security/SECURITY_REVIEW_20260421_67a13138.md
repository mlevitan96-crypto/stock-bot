### Summary

- **CRITICAL:** 0
- **HIGH:** 0
- **MEDIUM:** 5
- **LOW:** 4

**No confirmed secrets, credentials, or private keys were found committed to the repository.** The `.env` file is properly `.gitignore`d, and all API keys / tokens (`ALPACA_API_KEY`, `ALPACA_API_SECRET`, `UW_API_KEY`, `TELEGRAM_BOT_TOKEN`, `DASHBOARD_USER`, `DASHBOARD_PASS`) are loaded from environment variables or `.env` files at runtime. The `.env.example` file contains only empty placeholder keys.

No `verify=False` (TLS bypass) was found in any Python `requests` calls. All `requests.*()` calls include explicit `timeout` parameters.

No SQL injection, unsafe YAML loading (`yaml.load()` without `SafeLoader`), or `pickle.load()` on untrusted data was found.

---

### Recommendations

1. **Replace `exec(base64.b64decode(...))` with scp + remote execution.** Instead of encoding Python scripts as base64 strings and executing via `python -c 'exec(...)'`, copy the script to the droplet with `scp`/SFTP and run it directly. This eliminates an entire class of injection risks.

2. **Minimize `shell=True` usage.** Refactor `subprocess.run(cmd, shell=True)` calls to use list-form arguments (`subprocess.run(["cmd", "arg1", "arg2"])`) where possible. This prevents shell injection and is more portable.

3. **Replace `AutoAddPolicy` with known-hosts verification.** Use `paramiko.RejectPolicy()` or `paramiko.WarningPolicy()` and maintain a `known_hosts` file for the droplet. This prevents MITM attacks on the SSH connection.

4. **Add model integrity checks for `joblib.load()`.** Compute and verify SHA-256 checksums of model files before loading. Store expected hashes in a manifest file committed to the repo.

5. **Run `pip-audit` in CI.** Pin to latest patch versions of `requests`, `flask`, and `paramiko`. Set up a weekly or per-push `pip-audit` job to catch newly disclosed CVEs.

6. **Move hardcoded IPs to configuration.** Replace `104.236.102.57` references with the SSH config alias `alpaca` or an environment variable.
