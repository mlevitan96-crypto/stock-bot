# Order submission truth contract + full signal review — droplet commands

**DROPLET IS CANONICAL.** All artifacts are generated on the droplet; local runner fetches them.

---

## Phase 1 — Order submission truth contract

### 1) Deploy and restart (from local)

Ensure `main.py` with `SUBMIT_ORDER_CALLED` logging is on `origin/main`, then:

```bash
python scripts/deploy_submit_proof_and_run_contract_on_droplet.py
```

**Exact commands run on droplet:**
- `cd /root/stock-bot && git fetch origin && git reset --hard origin/main`
- `mkdir -p reports/order_review logs`
- `systemctl restart stock-bot` (or `pkill -f 'python.*main.py'` if no systemd)
- `python3 scripts/order_submission_truth_contract_on_droplet.py`

### 2) Wait ≥30 minutes (market hours if applicable)

### 3) Re-run contract to capture real counts (from local)

```bash
python scripts/deploy_submit_proof_and_run_contract_on_droplet.py --proof-only
```

Or:

```bash
python scripts/run_order_submission_truth_contract_via_droplet.py
```

**Required terminal output:**
- SUBMIT_ORDER_CALLED: YES/NO (count)
- submit_entry.jsonl lines: count
- Broker responses: success/fail counts
- FINAL VERDICT: one sentence

**Artifacts on droplet:**
- `logs/submit_order_called.jsonl` (exists if submit path is hit)
- `reports/order_review/submit_call_proof.md`
- `reports/order_review/ledger_vs_order_reconciliation.md`
- `reports/order_review/board_verdict.md`

## Phase 2 — Full signal review (expectancy gate)

From local (runs on droplet and fetches):

```bash
python scripts/run_deploy_submit_proof_and_full_signal_review_on_droplet.py --skip-deploy
```

**Exact command on droplet:**
- `python3 scripts/full_signal_review_on_droplet.py --days 7 --capture`

**Artifacts on droplet:**
- `reports/signal_review/signal_funnel.md`
- `reports/signal_review/signal_funnel.json`
- `reports/signal_review/top_50_end_to_end_traces.md`
- `reports/signal_review/multi_model_adversarial_review.md`

**Required terminal output (after Phase 2):**
- Dominant choke point: &lt;stage&gt;/&lt;reason&gt; (count, %)
- Expectancy: pre-adjust median vs post-adjust median
- % above MIN_EXEC_SCORE pre vs post
- Trades (paper): YES/NO (count)
- Join coverage: ledger %, snapshots %, UW %, adjustments %
